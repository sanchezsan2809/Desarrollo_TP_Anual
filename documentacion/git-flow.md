## Git-flow
Elegimos GitHub Flow como estrategia de trabajo debido a su simplicidad y seguridad a lo hora de hacer cambios.
Nos permite trabajar con ramas cortas asociadas a funcionalidades especificas, integrando cambios de forma continua mediante pull request.
Al no ser un proyecto grande no requiere multiples versiones de release, develop, etc. Priorizamos una metodología que facillite la colaboracion,
reduzca conflictos de integración y promueva la revisión temprana del codigo.

* **Entregable 1**: 
Se crearon las ramas factoryNotificacion, implementacionAgenda y healthCheckYEstructuraPorCapas para implementar las funcionalidades de mismo nombre, mientras que en paralelo, se abrieron Issues que denotaban la necesidad de estas funcionalidades. 
Una vez se llegó a un punto estable del desarrollo en cada rama, se abrieron Pull Requests para cada una, asociándolas a su respectivo Issue y posteriormente mergeándolas.
